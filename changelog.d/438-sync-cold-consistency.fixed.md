- **`clm slides sync` no longer reports a clean, committed, id-less deck as needing agent work (#438).** A split pair whose halves share no `slide_id` (a fully id-less or half-id'd deck) is routed off the keyed git-HEAD diff because that diff would double it — but when the working tree is byte-identical to git `HEAD` on both halves, the pair has not drifted since it was committed and is *consistent now*, not a cold start. The read surface (`report` / `task` / `accept` and the MCP `slides_sync_report` tool) treats it as the no-op a fully-id'd committed-unchanged pair already was, instead of surfacing a `mint` / `adopt` / `refuse` on every run. Cold-start candidacy still fires the moment such a pair is genuinely new (never committed) or has drifted from `HEAD`; a structural problem that was *committed* (e.g. mismatched ids on the two halves) is surfaced by `clm slides sync verify`, not by the drift report.
- **The agent read surface no longer gates cold-pair candidacy on an embedded API key (#438).** `report` / `task` / `accept` (and the MCP read tool) now always frame a genuinely-new id-less pair as a driveable `mint` / `adopt` *task* — the agent's validated answer is the verifier — rather than a dead-end `refuse` that depended on `OPENROUTER_API_KEY` being set in the environment. The key check survives only on the `autopilot` verb, which actually constructs the embedded model client.
