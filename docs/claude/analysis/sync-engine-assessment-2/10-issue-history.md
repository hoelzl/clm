## Summary

Mined the full sync bug history of hoelzl/clm via gh (30+ sync-titled issues, all pivotal bodies read, comments checked, plus the in-repo adversarial architecture assessment docs/claude/sync-engine-architecture-assessment.md merged via #288). The engine lineage: v1 = LLM-judge walker over (slide_id, role) pairs, updates-only (pre-#166); v2 = full classifier/plan/apply engine with sqlite watermark, shipped 2026-05-31 (#166 rewrite); a surface re-cut (not an engine rewrite) shipped as clm 1.16 on 2026-06-23 (epic #440: verb group report/verify/apply/task/accept/baseline/autopilot, models removed from engine path, watermark demoted to opt-in accelerator, git HEAD default baseline); a fourth state-model iteration (#448 git-committed per-slide ledger) was designed 2026-06-23 and is still landing. Defect cadence: 7 shipped + 1 open silent-drop/double bugs in the v2 engine's first 9 days (assessment §2.2), with documented re-fix chains (#216→#225→#226→e646fca5; #269 re-fixed #190's baseline coverage; #282 re-fixed positional assumptions in three earlier fixes at once), then a June 13-25 watermark/state bug family (#363/#364/#365/#366/#429/#430/#435/#477), then a post-re-cut dogfooding wave of UX gaps (#446/#447/#451) and two still-unmodeled-system findings (#443 keyed-companion drop, #501 sync blind to separated voiceover companions — OPEN).

Classified ~35 distinct defects into the five requested classes (primary classification; several are dual): (a) wrong/ad-hoc identity-alignment model ≈ 11 (#162, #166-gap, #216, #226, #282, #365, #403, #429, #438, #443, #242/#246/#247 vo_anchor edges); (b) engine didn't model part of the real system ≈ 7 (#198 tags channel, #269 neutral shared code bodies, #285/#289 tag×baseline×reorder cells, #190-item-2, #360 validate-vs-companions, #501 sync-vs-companions); (c) state management ≈ 10 (#202, #225, #363, #364, #366, #430, #435, #448, #477, #190-item-1 partial-apply atomicity); (d) UX/agent-comprehension ≈ 8 (#231, #364-msg, #430-trap, #438-false-alarm, #440 20-flags/7-jobs, #446, #447, #451); (e) ordinary implementation bug ≈ 4 (#377 JSON-quote truncation, #374-C judge response format, #458 comment-token threading, #281). Distribution verdict: only ~10% of the history is ordinary bugs; ~80% concentrates in model/state classes (a)+(b)+(c). That is strong evidence the pain is architectural, not sloppy coding — but the in-repo assessment (verdict "B: sound core, wrong in two identifiable places") argues, with executable probes, that the failures trace to two FIXABLE structural properties rather than a wrong core: (1) the cardinal invariant "propagate or alert, never silently drop" is enforced by hand-enumerated per-channel detectors with no completeness spine — every new channel (tags #198), baseline source (#269), or precondition interaction (reorder #282/#285) re-opens the class; the smoking gun is watermark_tag_map data recorded by _record_watermark that no code path ever read (P9); (2) the two-baseline duality (watermark vs git-HEAD) is implemented as parallel code paths whose information content is identical, and produced #269/#225/#226/P1 as pure plumbing-divergence bugs.

On the maintainer's two questions, the history supports: (1) salvageable — the body/structure channels of the v2 engine reached "prevented-or-always-surfaced" and every re-tested historical fix holds (P7/P8); what has NOT converged is the state layer (three watermark/baseline/ledger models in four weeks, and the watermark was per #440 "the source of nearly every active area:sync bug") and the scope boundary (companions excluded at every layer by an explicit 2026-06-02 design decision, split-voiceover-hardening.md §8, now being reversed via #501's inline→sync→extract composition). (2) The cell-referencing feels ad-hoc because it IS layered accretion: primary key (slide_id, role) with no occurrence/position component (#403 — every voiceover under one slide collapses to one key; the correct occurrence-qualified positional model, vo_anchor from PR #199, already existed in voiceover_tools.py and was deliberately not applied to the sync core), plus ordered-content-hash fallback for id-less localized cells (#365 — unsound under both-sided drift), plus positional stream alignment for cold start (#216/#438), plus raw sha256(strip) content identity (#429/#458 — a pure re-wrap reads as an edit). The assessment's crux finding stands: the {change-type × cell-class} matrix is essential complexity of bilingual mixed-identity sync, and id-less cells are genuinely ambiguous under reorder (alert-is-ceiling); but the {baseline-mode} axis, the per-channel enumeration, the absolute-path sqlite keying, and the fragmented identity fallbacks are accidental.

## Pain points

- Watermark/state layer is the #1 accidental-complexity hot-spot: 10+ issues in 4 weeks (#202 #225 #363 #364 #365-adjacent #366 #430 #435 #477 #448), three consecutive state models, absolute-path keys broken by worktrees AND subdirectory cwd AND renames (orphan rows), whole-deck granularity, and a self-baselining failure mode where advancing the watermark permanently banks silent drops. #440's own text: the watermark is 'the source of nearly every active area:sync bug'.
- No completeness spine for the propagate-or-alert invariant: detectors and fail-safes are enumerated per {channel × cell-class × baseline-source × precondition} cell by hand, so each new combination ships uncovered (#198 tags, #269 git-HEAD neutral bodies, #282 reorder interaction, #285/#289 tag×reorder/tag×git-HEAD, P9 recorded-but-never-read shared tags). #290's coverage-registry meta-test is the first structural counter-measure and arrived only after an external architecture review.
- Cell identity is 5 stacked ad-hoc mechanisms ((slide_id,role) key, ordered content hash, positional stream alignment, group-structural rebuild, sha256(strip)) rather than one reference model; the systematic occurrence-qualified anchor (vo_anchor) existed in the same codebase for a year of engine-time before #403 unified it into sync, and the (slide_id,role)-vs-bare-slide_id keying inconsistency between verify and plan/apply produced #443.
- The 4-file deck was never modeled: sync, validate-voiceover, and the build worker each independently rediscovered companion blindness (#501 #360 #324 #443). The 2026-06-02 decision to keep companions out of sync relied on a compensating detection gate that downstream repos didn't wire, while validator text and course instructions actively misdirected users to sync. N-file atomicity was deferred with 'design not started' and stayed that way for a month of daily use.
- Two-baseline duality as parallel code paths: git-HEAD and watermark baselines carry identical information but were plumbed separately, producing an entire bug family where one path lacked data the other had (#269 root cause verbatim; #225/#226 gate bugs; probe P1). Assessment recommends collapsing git-HEAD into a one-time watermark import (~2-3 days) — still not fully done; #458 shows even the hash function couldn't be changed atomically across sites.
- Agent-comprehension debt was structural until 2026-06-23: one leaf command, ~20 flags, 7 jobs, write-by-default, 4 embedded LLMs on the main path; #216's deck-doubling was prevented only by a missing API key ('safety-by-accident'). Post-re-cut, the residual (d) pain is information starvation in reports (empty conflict excerpts #451, bare 'N deferred' #231, misleading remedy strings #364/#438/#501-validator-text) — agents fall back to re-parsing raw files.
- Corpus-dependent correctness tests are skipped in CI (tests/slides/test_sync_corpus_mutation.py needs a local PythonCourses checkout, slow+integration) — #443 shipped through a green engine gate and was caught only while cutting a release; the highest-value oracles don't run where regressions are gated.
- LLM I/O robustness: free-form JSON from hosted judges truncated cells silently at unescaped quotes with 0 errors reported (#377); the same malformed-response class sometimes errored loudly and sometimes silently recovered a prefix — inconsistent failure handling on the materialization path (fixed via strict json_schema response_format in #374-C, but only after field data loss).
